1. Короткий опис продукту
Продукт - це спеціалізована онлайн-платформа, що дозволяє фанатам графічних новел швидко знаходити потрібні випуски за допомогою розширених фільтрів (видавництво, автор, серія). Основна проблема, яку вирішує сайт - складність пошуку рідкісних видань серед загальних книжкових магазинів. Користувач отримує можливість придбати комікс у кілька кліків, маючи актуальну інформацію про наявність та ціну.
2. Стейкхолдери та їхні інтереси
Покупець (Колекціонер): зацікавлений у наявності рідкісних видань, актуальних цінах та швидкості оформлення замовлення.
Менеджер магазину: хоче зручно керувати каталогом товарів, відстежувати залишки на складі та отримувати звіти про продажі.
Кур'єрська служба (Логіст): очікує вчасного надходження даних про замовлення та точних адрес доставки для виконання роботи.

3. Функціональні вимоги (FR)
FR-01: Система повинна дозволяти користувачу шукати комікси за назвою, автором або ISBN.
FR-02: Система повинна відображати статус наявності товару («в наявності», «передзамовлення», «немає на складі»).
FR-03: Система повинна забезпечувати можливість додавання обраних коміксів до віртуального кошика.
FR-04: Система повинна надавати форму для введення даних доставки та вибору методу оплати.
FR-05: Система повинна дозволяти користувачу створити особистий кабінет для відстеження статусу замовлення.
FR-06: Система повинна надсилати автоматичне повідомлення на email після успішного оформлення покупки.
4. Нефункціональні вимоги (NFR)
NFR-01 (Продуктивність): Результати пошуку в каталозі мають відображатися протягом не більше 1.5 секунди за умови одночасної роботи 200 користувачів.
NFR-02 (Безпека): Усі платежі повинні здійснюватися через захищений протокол HTTPS із підтримкою стандарту PCI DSS.
NFR-03 (Зручність): Користувач повинен мати можливість оформити покупку без обов'язкової реєстрації («гостьове замовлення»).
NFR-04 (Доступність): Сайт повинен мати мобільну адаптацію, що коректно працює на екранах з шириною від 320 пікселів.
NFR-05 (Надійність): Система повинна забезпечувати цілісність даних замовлень навіть у разі раптового розриву з'єднання з базою даних.
NFR-06 (Локалізація): Інтерфейс системи повинен підтримувати українську та англійську мови, перемикання між якими відбувається миттєво без перезавантаження головної сторінки.
5. Первинний Backlog (User Stories)
US-01: Як покупець, я хочу використовувати фільтри за видавництвами (Marvel, DC, Image), щоб швидше знайти комікси улюбленого всесвіту. Пріоритет: Must.
US-02: Як колекціонер, я хочу бачити фотографії обкладинки у високій якості, щоб оцінити стан видання перед купівлею. Пріоритет: Must.
US-03: Як постійний клієнт, я хочу зберігати товари у «Список бажань», щоб купити їх пізніше. Пріоритет: Should.
US-04: Як користувач, я хочу оплатити замовлення карткою безпосередньо на сайті, щоб пришвидшити процес отримання. Пріоритет: Must.
US-05: Як покупець, я хочу бачити орієнтовну дату доставки в моє місто, щоб спланувати отримання. Пріоритет: Should.

US-06: Як поціновувач, я хочу бачити блок «Схожі комікси» на сторінці товару, щоб знайти нові цікаві серії. Пріоритет: Could.
